# Experiments Protocol — Phases 0–3

> Status: Phase 1. Sub-agents A8 (eval/experiments) and A2/A9 (CLI/tests)
> own this document. Later phases extend with `experiments/PROTOCOL_CL.md`
> etc., but the contract below is frozen for Phases 0–3.

This document records the **experimental protocol** that every
experiment runner under `experiments/` must follow, plus the
canonical mapping from each paper claim to the script that tests
it. It exists so:

1. Reviewers can locate the exact run that supports any claim in
   the paper.
2. The CI hooks (`pytest -m cross_backend`, `make audit`) can
   validate that the experiments are still runnable as the library
   evolves.
3. New experiment scripts (Phase 4–12) follow the same shape.

---

## 1. Layout

```
experiments/
├── __init__.py
├── PROTOCOL.md         # this file
├── REPRODUCE.md        # human-friendly quickstart
├── run_exp_a_*.py      # one file per experiment, ≤ 250 LOC
├── run_exp_b_*.py
├── run_exp_optuna_search.py
└── …
```

## 2. Experiment contract

Each `run_exp_*.py` script **must**:

- Be runnable as `python -m experiments.run_exp_<id>_<topic>` and
  via Typer (`pjepa run exp-<id>`) in Phase 4+.
- Set all RNGs via `pjepa.utils.seeding.set_global_seed(seed)`.
- Accept `--out / -o` for the output directory and produce
  `results/<dataset>/<exp_id>/metrics.json` and `report.md`.
- Log per-step using `pjepa.logging_setup.configure_logging()`.
- Validate inputs via `pjepa.exceptions.ConfigError` /
  `DataError` — bare `print` is forbidden.

## 4. Statistical procedure

For every published table or figure:

1. Run with **5 different seeds** (`0..4`), defaulting to seed `0`.
2. Compute paired bootstrap 95% CIs via
   `pjepa.eval.bootstrap.paired_bootstrap_ci`.
3. Apply Bonferroni correction when comparing ≥ 3 conditions.
4. Report **mean ± std** and the raw per-seed values in
   `results/.../seeds.json`.

## 5. Cross-backend check

Every experiment must declare a backend string at the top of its
output (`mps`, `cuda`, `cpu`) using `pjepa.hardware.detect_backend()`.

## 6. Reproducibility checklist

- [ ] `pjepa doctor` returns GREEN for all probes relevant to
      the experiment.
- [ ] Output files include a `meta.json` with: seed, commit SHA,
      environment hash, device name, torch version, timestamps.
- [ ] `pytest -m cross_backend -q` passes locally.
- [ ] `pip-audit --strict` reports no high-severity findings.

## 7. Lifecycle

| Phase | What changes |
|---|---|
| 0–3 | Scaffold only; augmentations + perf infra (this document). |
| 4+ | Add `experiments/PROTOCOL_CL.md`, `PROTOCOL_OG.md` etc. for the continual-learning and OGB-arxiv suites. Do not edit this file from a Phase 4+ runner. |
